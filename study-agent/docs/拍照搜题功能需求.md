# 拍照搜题功能需求文档

## 一、功能概述

用户通过相机拍摄试卷照片，系统自动识别题目、学生信息，判断答题正确性，并将答题记录与知识点关联入库。

## 二、核心流程

```
拍照上传 → 图像预处理 → OCR识别 → 学生定位 → 题目解析 → 答案判定 → 知识点关联 → 入库
```

## 三、功能拆分

### 3.1 图像采集与预处理

| 功能 | 描述 |
|------|------|
| 拍照上传 | 支持相机拍摄或相册选择，图片格式 JPG/PNG |
| 图像矫正 | 自动检测表格边界，校正倾斜、畸变 |
| 去噪增强 | 灰度化、二值化、去阴影，提升 OCR 识别率 |
| 区域分割 | 将试卷图片切割为学生信息区和题目区域 |

### 3.2 学生信息识别

| 字段 | 说明 |
|------|------|
| 老师姓名 | 试卷上标注的教师姓名 |
| 学生姓名 | 手工填写，需要 OCR 识别 |
| 学号 | 手工填写，数字+字母组合 |

**定位策略**：
1. 优先检索固定位置（如试卷顶部姓名栏）
2. OCR 识别后匹配 `t_student` 表中的学生信息
3. 若未找到匹配学生，记录为"未注册学生"并允许新建

### 3.3 题目解析

| 步骤 | 说明 |
|------|------|
| 题干识别 | 提取题目文本内容 |
| 选项识别 | 区分 A/B/C/D 等选项 |
| 答案提取 | 识别学生手写/印刷的答案 |
| 题号识别 | 确定每道题对应的题号 |

**题目格式支持**：
- 选择题（单选/多选）
- 判断题
- 填空题
- 简答题（识别关键字）

### 3.4 答案判定

| 逻辑 | 处理方式 |
|------|----------|
| 选择题 | 比对学生答案与标准答案，区分大小写 |
| 判断题 | T/F 或 √× 统一映射 |
| 填空题 | 关键词匹配，支持部分得分 |
| 简答题 | 关键字匹配，计算得分比例 |

### 3.5 知识点关联

**来源**：
- `t_question.knowledge_point_ids` 题目预设的知识点
- AI 分析题目内容后补充推荐的知识点

**关联逻辑**：
1. 根据题目内容匹配 `t_knowledge_point` 表
2. 计算该学生对该知识点的掌握度变化
3. 更新 `t_student_answer.knowledge_point_id` 和 `mastery_level`

### 3.6 数据入库

| 表名 | 操作 |
|------|------|
| `t_student_answer` | 新增答题记录 |
| `t_question` | 新增/更新题目（如遇新题） |
| `t_student_knowledge_mastery` | 更新知识点掌握度 |

## 四、API 设计

### 4.1 拍照搜题接口

```
POST /api/exam/upload
Content-Type: multipart/form-data

请求参数：
- image: 文件（图片）
- classId: Long（可选，限定班级）

响应：
{
  "success": true,
  "data": {
    "studentId": 123,
    "studentName": "张三",
    "totalQuestions": 20,
    "correctCount": 15,
    "accuracyRate": 75.0,
    "answers": [
      {
        "questionId": 456,
        "studentAnswer": "A",
        "correctAnswer": "A",
        "isCorrect": true,
        "knowledgePoints": ["KP001", "KP003"]
      }
    ]
  }
}
```

### 4.2 批量录入模式（可选）

```
POST /api/exam/batch
Content-Type: multipart/form-data

请求参数：
- images: 文件数组（多张试卷）

响应：返回每张试卷的识别结果列表
```

## 五、技术方案

### 5.1 OCR 方案

| 方案 | 适用场景 | 成本 |
|------|----------|------|
| 百度 OCR | 通用文字识别，精度高 | API 调用计费 |
| Tesseract OCR | 本地部署，免费 | 精度稍低 |
| 阿里 OCR | 教育场景优化 | API 调用计费 |

**推荐**：使用百度 OCR 或阿里 OCR教育版，支持表格识别和手写体识别。

### 5.2 AI 题目分析（可选）

当题目无法直接匹配标准答案时，调用 AI 分析：
- 题目知识点提取
- 答案正确性判断
- 相似题目推荐

### 5.3 图像预处理流程

```
原图 → 灰度化 → 去噪 → 二值化 → 边缘检测 → 透视变换 → 表格分割 → 区域裁剪
```

## 六、数据模型扩展

### 6.1 新增表：`t_exam_record`（试卷记录）

| 字段 | 类型 | 说明 |
|------|------|------|
| id | Long | 主键 |
| student_id | Long | 学生ID |
| teacher_name | String | 老师姓名 |
| class_id | Long | 班级ID |
| total_questions | Integer | 总题数 |
| correct_count | Integer | 正确数 |
| image_url | String | 原始图片URL |
| status | String | pending/completed/failed |
| create_time | DateTime | 创建时间 |

### 6.2 修改表：`t_student_answer`（已有）

新增字段：
- `exam_record_id`: Long - 所属试卷记录ID

## 七、错误处理

| 场景 | 处理方式 |
|------|----------|
| 图片模糊无法识别 | 返回"识别失败，请重新拍照" |
| 学生信息匹配失败 | 允许手工输入或创建新学生 |
| 题目匹配失败 | 标记为"新题"，进入题库待审核 |
| 部分题目识别失败 | 仅入库已识别题目，给出识别率 |

## 八、非功能性需求

| 指标 | 要求 |
|------|------|
| 图片大小 | ≤ 10MB |
| 识别耗时 | < 5秒（不含AI分析） |
| 文字识别率 | ≥ 95%（印刷体） |
| 支持图片格式 | JPG, PNG, BMP |
| 并发支持 | 支持 10+ 并发请求 |

## 九、后续扩展

1. **错题本**：自动收集答错题目，生成错题本
2. **学情分析**：基于答题数据生成学生知识点掌握图谱
3. **智能推荐**：根据薄弱知识点推荐练习题
4. **批改报告**：生成班级整体答题情况报告
