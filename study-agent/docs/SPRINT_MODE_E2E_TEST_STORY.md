# 冲刺模式 E2E 测试用户故事

## 测试目标

验证冲刺模式完整流程：登录 → 进入冲刺 → 答题 → 结算 → 奖励发放 → 错题记录

---

## 用户故事 1：完整冲刺流程

### 故事描述
作为学生用户，我希望能完成一次完整的冲刺模式答题，查看结算页面，验证奖励和错题记录是否正确。

### 前置条件
- 测试环境 MySQL 已启动
- 后端服务已启动 (`mvn spring-boot:run`)
- 前端服务已启动 (`npm run dev`)
- 数据库中有可用的题目数据（科目：英语，ID=3）

### 测试步骤

#### 1. 登录
- 访问登录页面 `/login`
- 输入用户名 `testuser`
- 输入密码 `testpass123`
- 点击登录按钮
- 验证跳转到首页 `/student/home`

#### 2. 进入冲刺模式
- 在首页点击「冲刺模式」入口
- 验证跳转到冲刺页面 `/student/sprint`

#### 3. 选择科目并开始
- 页面加载后验证显示科目选择
- 选择「英语」（subjectId=3）
- 点击「开始挑战」按钮
- 验证页面进入答题状态
- 验证显示 60 秒倒计时

#### 4. 答题（答对 3 题，答错 1 题）
- 等待题目加载
- 第一题：选择正确答案（记录答案标识如 "A"）
- 等待下一题
- 第二题：选择正确答案
- 等待下一题
- 第三题：故意选择错误答案
- 等待下一题
- 第四题：选择正确答案
- 继续答题直到时间耗尽或手动结束

#### 5. 结算验证
- 等待结算页面显示
- 验证显示本次成绩：总分数、答题数、正确数、错误数、正确率、最大连击
- 验证显示奖励：金币、经验、钻石（如有）
- 验证显示是否破纪录

#### 6. 再来一局
- 点击「再来一局」按钮
- 验证重新进入答题状态
- 快速手动结束（点击放弃按钮）

#### 7. 返回首页
- 点击「返回首页」按钮
- 验证跳转到首页 `/student/home`

## 用户故事 2：金币和经验增长验证

### 故事描述
作为学生用户，我希望能验证冲刺模式获得的金币和经验是否正确累加到我的账户。

### 前置条件
完成故事 1 的步骤 1-7，记录结算时的金币和经验奖励数值。

### 测试步骤

#### 1. 记录冲刺前余额
- 进入「我的」页面 `/student/profile`
- 记录当前金币余额 `gold_before`
- 记录当前经验值 `exp_before`

#### 2. 完成一次冲刺
- 执行用户故事 1 的步骤 2-5
- 记录本次获得的金币 `gold_earned`（从结算页面）
- 记录本次获得的经验 `exp_earned`

#### 3. 验证余额增长
- 返回「我的」页面
- 验证金币余额 = `gold_before` + `gold_earned`
- 验证经验值 = `exp_before` + `exp_earned`

### 验收标准
- 金币余额正确累加
- 经验值正确累加
- 如果连击达到 3/5/10/20，钻石也应正确累加

## 用户故事 3：错题本记录验证

### 故事描述
作为学生用户，我希望能验证冲刺模式中答错的题目是否被正确记录到错题本。

### 前置条件
- 完成一次冲刺，其中至少答错 1 题
- 记录答错的题目 ID（从结算页面的答题记录中）

### 测试步骤

#### 1. 进入错题本
- 在首页点击「错题本」入口（路径可能是 `/student/wrong-answers`）
- 验证页面加载成功

#### 2. 验证错题记录
- 在错题列表中查找冲刺模式答错的题目
- 验证题目内容与冲刺时一致
- 验证记录的答案为错误答案
- 验证正确答案已记录

### 验收标准
- 冲刺中答错的题目出现在错题本中
- 错题记录包含：题目内容、用户答案、正确答案

## 用户故事 4：冲刺模式题目不重复

### 故事描述
作为学生用户，我希望在一次冲刺中遇到的题目都不会重复。

### 前置条件
数据库中该科目至少有 10 道题。

### 测试步骤

#### 1. 开始冲刺
- 进入冲刺模式
- 选择科目（英语）
- 开始冲刺

#### 2. 验证题目不重复
- 每答一题，记录题目 ID
- 验证同一题目 ID 不会出现两次
- 持续答题直到题目用尽或时间结束

### 验收标准
- 所有已答题目 ID 集合无重复
- 当题库题目用尽时，系统提示或正常结束

## 用户故事 5：倒计时与自动结束

### 故事描述
作为学生用户，我希望冲刺模式能在 60 秒后自动结束，显示结算页面。

### 测试步骤

#### 1. 开始冲刺
- 进入冲刺模式
- 选择科目
- 开始冲刺

#### 2. 观察倒计时
- 验证倒计时从 60 开始
- 每秒减少 1
- 倒计时到 0 时自动触发结束

#### 3. 验证自动结算
- 验证页面自动跳转到结算页面
- 验证结束原因显示为「时间到」或类似标识

### 验收标准
- 60 秒倒计时正常工作
- 时间耗尽自动结束并显示结算

---

## 用户故事 6：连击与奖励机制

### 故事描述
作为学生用户，我希望连击越高，获得的奖励倍率越高。

### 前置条件
数据库中该科目至少有 5 道简单题可连续答对。

### 测试步骤

#### 1. 连续答对
- 开始冲刺
- 连续答对 5+ 题（确保每题答案正确且快速）
- 记录连击数变化

#### 2. 验证连击奖励
- 观察每题得分是否随连击增加
- 验证 3 连击时获得 1 钻石
- 验证 5 连击时获得 5 钻石
- 验证 10 连击时获得 20 钻石
- 验证 20 连击时获得 50 钻石

#### 3. 中断连击
- 故意答错 1 题
- 验证连击重置为 0
- 验证后续得分按新连击计算

### 验收标准
- 连击正确累计
- 答错后连击归零
- 钻石奖励按连击档位发放
